installation

                        
                            # Install from nuget.org
                            dotnet tool install -g --no-cache --ignore-failed-sources spex --no-cache

                            # Upgrade to latest version from nuget.org
                            dotnet tool update -g --no-cache --ignore-failed-sources spex

                            # Install a specific version from nuget.org
                            dotnet tool install -g --no-cache --ignore-failed-sources spex --version 1.0.x

                            # Uninstall
                            dotnet tool uninstall -g spex

If you don't have .NET Core 3.1 installed, download and install it.

If the Tool is not accessible post installation, add %USERPROFILE%\.dotnet\tools to the PATH env-var.

usage

 spex [MyProject.Spex.Settings.json]

  • If the specified path or the current-dir contains a valid Settings file, the tool uses/creates the Spex folder under that path (for processing inputs/outputs); else, it falls back to the Pefx folder under My Documents
  • If no argument is supplied, Spex.Settings.json is used as input by default
  • Set QuiteMode to true in the Settings file to run once and exit

  • Enter r:path-to-bdd-test-project to execute the sync to AzDevOps
  • GenerateTestCaseUserStoryAssociationsMatrix
  • GenerateTestCasesReport
  • Enter d:path-to-bdd-test-project to demo/dry-run the sync to AzDevOps
  • Enter c to clear the console
  • Enter q to quit
  • Enter ? to print this help
